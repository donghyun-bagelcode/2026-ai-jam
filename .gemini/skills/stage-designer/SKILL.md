# Stage Designer — 스테이지 설계 및 검증

## 목적

World 1의 10개 스테이지를 기획서 정책에 맞게 설계한다.

## W1 설계 정책 (기획서 기준)

- 맵/시작/포탈/키 위치: 스테이지별 고정
- Stage 1~3: 시작 위치 하단 중앙
- Stage 4~10: 시작 위치 개별 고정
- 키 배치 금지: 시작 타일, 포탈 타일

## 난이도 곡선

- Stage 1~3: 기본 조작/룰 학습 (M = 3~5)
- Stage 4~6: 경로 선택 분기 도입 (M = 5~8)
- Stage 7~8: 가벼운 트릭 — 우회, 역순 경로 (M = 7~10)
- Stage 9~10: 월드1 종합 문제 (M = 9~12)

## 설계 검증 체크리스트

- [ ] 키 3개가 시작/포탈 타일이 아닌 곳에 배치되었는가?
- [ ] 클리어 가능한가? (M값이 존재하는가?)
- [ ] M값이 난이도 곡선에 맞는가?
- [ ] 3별 경로가 비자명한가? (너무 뻔하지 않은가?)
- [ ] 1별 경로(비효율 경로)도 존재하는가?

## 스테이지 데이터 포맷 (초안)

```json
{
  "id": 1,
  "width": 7,
  "height": 7,
  "walls": [[row, col], ...],
  "start": [row, col],
  "keys": [[row, col], [row, col], [row, col]],
  "portal": [row, col],
  "minMoves": 4
}
```

## 구현 시 보완할 내용

- [ ] 실제 스테이지 10개 설계
- [ ] min-move-solver 연동하여 M값 자동 검증
- [ ] 스테이지 시각화/프리뷰 도구
