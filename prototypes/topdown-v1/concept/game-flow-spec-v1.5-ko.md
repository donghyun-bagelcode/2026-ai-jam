# 슬라이드 퍼즐 게임 플로우 명세서 v1.5

- 작성일: 2026-03-05
- 버전: v1.5
- 목적: 앱 실행부터 종료까지 사용자 플로우/화면/팝업/예외처리 기준 정리
- 기준 문서:
  - [슬라이드 퍼즐 게임 통합 기획서 v1.5](./game-design-master-v1.5-ko.md)
  - [월드 1 스테이지 레이아웃 v1.5](./stage-layouts-v1-ko.md)

## 1. 범위

- 출시 플레이 범위: `World 1 Basic(10) + Hard(10)`
- World 2~5: 월드 리스트에 잠금 상태로 노출, 플레이 불가
- 입력 플랫폼: 모바일 스와이프 우선

## 2. 전체 플로우(요약)

1. 앱 아이콘 터치
2. 스플래시/로딩 화면
3. 월드 리스트 화면
4. World 1 진입
5. World 1 스테이지 선택 화면(Basic/Hard)
6. 스테이지 플레이 화면
7. 결과 팝업
8. 다음 스테이지 진입 또는 스테이지 선택 복귀
9. 앱 종료/백그라운드 복귀

## 3. 화면별 명세

## 3.1 스플래시/로딩

- 표시 요소:
  - 게임 타이틀
  - 로딩 바
  - 배경 아트
- 완료 조건: 필수 리소스 로드 완료
- 완료 액션: 월드 리스트로 자동 전환

## 3.2 월드 리스트

- 표시 요소:
  - World 1(활성)
  - World 2~5(잠금)
- 버튼:
  - World 1 카드 탭 -> World 1 스테이지 선택 화면
  - World 2~5 카드 탭 -> `Coming Soon` 토스트(권장) 또는 무반응 정책 중 택1
- 출시 기준 권장 동작:
  - 무반응 대신 `잠금 안내 토스트 1~1.5초`로 피드백 제공

## 3.3 World 1 스테이지 선택

- 상단:
  - 좌측 상단: 백 버튼(월드 리스트 복귀)
  - 중앙: `World 1` + 현재 모드(`Basic` 또는 `Hard`)
  - 우측 상단: 캐릭터 선택 버튼
- 모드 전환:
  - `Basic` 탭(항상 활성)
  - `Hard` 탭(잠금/해금 상태)
- 별 정보:
  - 현재 모드 별 합계 표시(`x/30`)
- 스테이지 그리드:
  - 1~10
  - 클리어 별(1~3) 표시
  - 잠금 스테이지는 잠금 아이콘 표시
- 캐릭터 위치:
  - 마지막 클리어 스테이지 타일 위 표시
  - 첫 진입(클리어 이력 없음) 시 Stage 1 위치

### 3.3.1 Hard 잠금/해금

- 잠금 조건: Basic 10개 전부 3별 미달
- 해금 조건: Basic 10개 전부 3별
- 잠금 상태 탭 터치 시:
  - 진행률 안내 팝업(예: `27/30`, `남은 3별 스테이지`) 권장

### 3.3.2 캐릭터 선택 팝업

- 표시 요소:
  - SD 캐릭터 리스트
  - 현재 선택 체크 표시
  - 닫기(X)
- 동작:
  - 캐릭터 탭 시 즉시 선택 반영
  - X 탭 시 변경 없이 닫기

## 3.4 스테이지 플레이

- 맵 구성:
  - 이동 타일 / 벽 / 키 / 포탈 / 기믹 타일
  - 맵 규격: 내부 `5x8`, 전체 `7x10`
- HUD:
  - 상단: 키 획득 상태(0/3, 1/3, 2/3, 3/3)
  - 하단: 이동 횟수(Moves)
- 인풋:
  - 스와이프 4방향
  - 슬라이드 중 입력 잠금(추가 입력 무시)

### 3.4.1 인게임 공통 버튼(추가)

- 우상단 또는 좌상단: 일시정지 버튼(필수 권장)
- 일시정지 팝업:
  - `재개`
  - `다시 시작`
  - `스테이지 선택으로`

## 3.5 결과 팝업(클리어)

- 표시 요소:
  - 획득 별(1~3)
  - 이동 횟수
  - Best 기록(선택)
- 버튼:
  - `Retry` (같은 스테이지 재시작)
  - `Next` (다음 스테이지)
  - `X` (스테이지 선택 복귀)

### 3.5.1 Stage 10 예외 처리

- Basic Stage 10에서 `Next` 탭 시:
  - Hard 해금 조건 충족 시: `Hard 오픈 연출` 후 Hard Stage 1 진입 버튼 제공
  - 미충족 시: 스테이지 선택 복귀 + 안내 팝업
- Hard Stage 10에서 `Next` 탭 시:
  - `World 1 완료` 팝업 + 스테이지 선택 복귀

## 4. 상태 전이 표

| 현재 상태 | 이벤트 | 다음 상태 | 비고 |
|---|---|---|---|
| Splash | 로딩 완료 | WorldList | 자동 전환 |
| WorldList | World1 탭 | StageSelect(W1,Basic) | |
| WorldList | World2~5 탭 | WorldList | 잠금 안내 토스트 |
| StageSelect | Stage 탭(해금) | InGame | |
| StageSelect | Stage 탭(잠금) | StageSelect | 잠금 안내 |
| StageSelect | 캐릭터 버튼 | CharacterPopup | |
| CharacterPopup | 캐릭터 선택 | StageSelect | 즉시 반영 |
| InGame | 클리어 | ResultPopup | |
| InGame | 일시정지 | PausePopup | |
| ResultPopup | Retry | InGame(동일 스테이지) | 상태 완전 초기화 |
| ResultPopup | Next | InGame(다음 스테이지) 또는 StageSelect | Stage10 예외 적용 |
| ResultPopup | X | StageSelect | 저장 완료 후 이동 |

## 5. 저장/복구 타이밍

- 저장 시점:
  - 스테이지 클리어 확정 직후(결과 팝업 오픈 직전)
  - 캐릭터 선택 변경 시
  - 모드/스테이지 해금 상태 변경 시
- 저장 항목:
  - 스테이지별 최고 별
  - 클리어 여부
  - 모드 해금(Basic/Hard)
  - 캐릭터 선택값
  - 마지막 클리어 스테이지
- 복구 정책:
  - 앱 재실행 시 월드 리스트부터 시작
  - 인게임 중 강종 복구는 v1.5 범위 외(후속)

## 6. 예외사항/안전장치

1. 슬라이드 중 다중 입력
- 정책: 입력 잠금, 애니메이션 완료 후 해제

2. 결과 팝업 연타/중복 탭
- 정책: 첫 입력만 처리(버튼 디바운스)

3. 잠금 월드/잠금 스테이지 탭
- 정책: 상태 변화 없음 + 안내 메시지

4. 백그라운드 전환
- 정책: 즉시 일시정지 상태로 전환

5. 로컬 저장 실패
- 정책: 에러 토스트 노출 + 재시도 버튼 제공

## 7. UI 체크리스트

1. 월드 리스트
- World1 활성, World2~5 잠금 시각 구분

2. 스테이지 선택
- Basic/Hard 탭 상태 명확
- 별 합계(`x/30`) 노출
- 스테이지 잠금/해금 가시성

3. 인게임
- 키/이동 횟수 HUD 항상 고정
- 일시정지 접근 가능

4. 결과 팝업
- 별 결과 + Retry/Next/X
- Stage10 예외 분기 확인

## 8. QA 필수 시나리오

1. 첫 실행부터 Basic Stage1 클리어까지 정상 진행
2. Basic 10개 3별 달성 시 Hard 해금
3. Hard Stage 10 클리어 후 완료 팝업 분기
4. 잠금 월드 탭 시 비정상 진입 없음
5. 결과 팝업에서 Retry/Next/X 저장 정합성 유지
6. 앱 백그라운드 전환 후 입력 잠금/해제 정상

## 9. 결정 필요 항목(최종 확정용)

1. 잠금 월드 탭 동작
- A안: 완전 무반응
- B안(권장): `Coming Soon` 토스트

2. Basic Stage10 클리어 시 `Next` 우선 동작
- A안: Hard 해금 안내 우선
- B안: 스테이지 선택 복귀 우선

3. 강종 복구 정책(v1.5)
- A안: 항상 스테이지 선택 복귀
- B안: 마지막 인게임 상태 복구(후속)
