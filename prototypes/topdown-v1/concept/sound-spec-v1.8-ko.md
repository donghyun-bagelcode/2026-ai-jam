## 1. 범위

- 출시 플레이 범위: `World 1 Basic(10) + Hard(10)`
- 포함: BGM, UI SFX, 인게임 코어 SFX, 하드 기믹 SFX, 결과/보상 SFX
- 제외: World 2~5 전용 테마 BGM/SFX(후속 업데이트)

## 2. 오디오 방향성

- 기본 톤: 감성적/부드러운 분위기 + 퍼즐 가독성 우선
- 믹스 원칙:
  - BGM은 플레이 집중을 해치지 않는 중저밀도 편성
  - 핵심 상태 변화(키 획득, 포탈 활성, 클리어)는 1회 청취로 인지 가능해야 함
  - 제자리 스와이프는 Moves 미증가 정책에 맞춰 과한 피드백 금지

3. 폴리시(P1/P2): 나머지 항목 순차 제작

## 6. 파일 네이밍/구조

- 경로: `assets/sounds/`
- 형식: 작업본 `wav`, 빌드본 `mp3` 또는 `ogg`
- 네이밍 규칙(권장):
  - `bgm-title-loop-01.mp3`
  - `bgm-ingame-basic-loop-01.mp3`
  - `sfx-ui-tap-01.mp3`
  - `sfx-game-key-pickup-01.mp3`
  - `sfx-hard-warp-inout-01.mp3`

## 7. 구현/믹스 가이드

- 볼륨 밸런스(권장 시작점):
  - BGM: -18 ~ -14 LUFS
  - 핵심 SFX(키/포탈/클리어): BGM 대비 +4~+7dB
  - 일반 UI SFX: BGM 대비 +1~+3dB
- 동시 재생 제한:
  - 같은 이벤트 SFX는 100ms 이내 중복 트리거 방지
  - 슬라이드 루프(SFX-06)와 충돌(SFX-07)은 우선순위로 덕킹 처리
- 규칙 연동:
  - 제자리 스와이프는 Moves 증가가 없으므로 `SFX-05`는 저강도 버전 또는 미재생 처리
  - 마지막 키 획득 슬라이드에서 즉시 클리어 시 `SFX-09 -> SFX-11` 연쇄 재생

## 8. QA 체크리스트

1. 핵심 상태 변화(키 획득/포탈 활성/클리어)가 소리만으로 구분되는가
2. 하드 기믹(오순서/일방통행/워프)에서 오디오 피드백이 정확히 구분되는가
3. 버튼 연타 시 SFX 중복 폭주가 없는가
4. BGM 루프 포인트에서 이음새(클릭/팝 노이즈)가 없는가
5. 저사양/무음 모드에서도 UX가 깨지지 않는가
