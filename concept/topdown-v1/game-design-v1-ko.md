# 슬라이드 퍼즐 게임 기획서 v1

- 작성일: 2026-03-04
- 버전: v1.0 (룰 기반 확정본)
- 범위: World 1 구현 기준

## 1. 게임 개요

본 게임은 **슬라이드 이동 기반 경로 최적화 퍼즐**이다.  
플레이어는 상/하/좌/우 스와이프로 캐릭터를 이동시키며, 캐릭터는 입력한 방향으로 벽에 부딪힐 때까지 미끄러지듯 이동한다.

스테이지 목표는 다음 2단계다.

1. 키 3개를 모두 수집
2. 활성화된 포탈에 도달하여 클리어

핵심 재미는 실패 압박이 아닌, **최소 이동(슬라이드 횟수)으로 클리어하는 효율 도전**이다.

## 2. 코어 규칙

### 2.1 이동 규칙

- 입력: 모바일 스와이프(상/하/좌/우)만 지원
- 캐릭터는 입력 방향으로 연속 이동
- 벽 또는 이동 불가 타일을 만나면 정지
- 슬라이드 1회 = 이동 횟수 1로 카운트

### 2.2 오브젝트 규칙

- 키는 총 3개
- 캐릭터가 슬라이드 중간에 키 타일을 지나가면 즉시 획득
- 포탈은 스테이지 입장 시 비활성 상태
- 키 3개 획득 시 포탈 활성화
- 비활성 포탈은 벽처럼 취급(통과/정지 불가)

### 2.3 클리어/실패 규칙

- 활성 포탈 타일에 캐릭터가 도달하면 스테이지 클리어
- 실패 조건 없음(언제든 클리어 가능)

## 3. 별(Star) 평가 시스템

- `M`: 해당 플레이(배치 기준)의 최소 이동 횟수
- `P`: 플레이어 실제 이동 횟수

평가 규칙:

- `P == M` => 별 3개
- `P <= M + 2` => 별 2개
- 그 외 => 별 1개

## 4. 스테이지/월드 구조

### 4.1 기본 구조

- 1개 월드 = 10개 스테이지
- 스테이지 번호: 1 ~ 10
- 직전 스테이지를 클리어해야 다음 스테이지 해금
- 클리어한 스테이지는 언제든 재도전 가능

### 4.2 월드 해금 규칙

- 월드 누적 별이 20개 이상이면 다음 월드 해금
- 월드 누적 별 = 각 스테이지의 **최고 별 기록 합계**

예시:

- Stage 1: 1회차 2별, 2회차 1별 -> 최종 2별 유지
- Stage 2: 1회차 1별, 2회차 3별 -> 최종 3별 갱신

## 5. World 1 확정 정책 (튜토리얼 월드)

World 1은 학습/튜토리얼 성격을 위해 난이도 제어를 우선한다.

- 맵 형태: 스테이지별 고정
- 시작 위치: 스테이지별 고정 1개
- 포탈 위치: 스테이지별 고정
- 키 3개 위치: 스테이지별 고정
- 키 위치 예외: 시작 타일, 포탈 타일에는 배치 금지

참고 정책:

- Stage 1~3은 학습을 위해 시작 위치를 하단 중앙으로 우선 배치
- Stage 4~10은 퍼즐 의도에 맞게 시작 위치를 개별 고정

## 6. 저장 정책

- 저장 방식: LocalStorage
- 저장 대상:
  - 스테이지 클리어 여부
  - 스테이지별 최고 별 기록
  - 월드 누적 별 합계
  - 해금 상태(스테이지/월드)

## 7. 난이도/설계 방향

### 7.1 World 1 목표

- 슬라이드 규칙 이해
- 벽 반사/경로 압축 감각 학습
- 별 3개를 위한 최소 경로 탐색 동기 부여

### 7.2 스테이지 난이도 곡선(권장)

- Stage 1~3: 기본 조작/룰 학습
- Stage 4~6: 경로 선택 분기 도입
- Stage 7~8: 가벼운 트릭(우회, 역순 경로 유도)
- Stage 9~10: 월드1 종합 문제

## 8. 향후 확장 로드맵 (초안)

- World 2: 키 순서 시스템(1 -> 2 -> 3)
- World 3: 동적 출구 시스템(키 완료 후 후보 중 출구 생성)
- World 4: 일방 통행 타일
- World 5: 텔레포트 타일

또한, 키 랜덤 배치는 World 2+에서 재도입을 검토한다.

## 9. 구현 체크리스트 (v1)

1. 슬라이드 이동/충돌 처리
2. 키 획득 및 포탈 활성화 상태 머신
3. 최소 이동 계산 로직(`M`) 연동
4. 별 산정 및 최고 기록 갱신
5. 스테이지 순차 해금
6. 월드 누적 별 계산 및 다음 월드 해금
7. LocalStorage 저장/복원
